# 에러 처리/정합성

## 1. 외부 시스템은 오류가 발생해도 롤백되지 않는다

- 고객이 PG사에서 결제를 마치고 우리 서버에서 승인까지 진행되어 실제 카드 결제는 완료되었으나, 이후 후속 처리 과정에서 오류가 발생하여 주문이 **PENDING(대기)** 상태로 남는 경우가 존재한다.
- 외부 시스템 연동은 로컬 데이터베이스(DB) 트랜잭션처럼 자동으로 롤백 처리가 되지 않기 때문이다.
- 외부는 롤백되지 않으므로 부분 실패는 보상하고, 결과를 알 수 없는 모호한 상태는 사후에 수렴시켜 **결과적 정합성**을 맞춰야 한다.
- 단일 트랜잭션을 사용할 수 없으므로(아래에서 이유 확인) 프로세스 각 단계마다 상태 값을 기록하여 관리해야 한다.
- 이러한 흐름 제어 방식을 **사가 패턴(Saga Pattern)** 이라고 부르며, 크게 두 가지 방식으로 나뉜다.
  - **코레오그래피(Choreography)**:
    - 중앙 조정자 없이 각 서비스가 이벤트를 발행·구독하며 자신의 로컬 트랜잭션을 스스로 이어나가는 방식이다.
    - 서비스 간 결합도는 낮지만 전체 흐름을 한눈에 파악하기 어렵다.
  - **오케스트레이션(Orchestration)**:
    - 하나의 조정자가 각 단계를 순서대로 호출하고 그 결과에 따라 상태를 관리하는 방식이다.
    - 흐름이 한 곳에 모여 있어 파악과 관리가 쉽다.
- 마이크로서비스 환경이 아니더라도, 로컬 애플리케이션 내부에서 하나의 조정자가 여러 외부 단계를 순차 호출하고 상태를 관리하는 **오케스트레이션 방식**으로 사가 패턴을 적용할 수 있다.

### 1.1. 외부 시스템 연동이 @Transactional 블록으로 묶여 있는 경우

- **커넥션 풀 고갈**: 외부 PG사나 외부 포인트 시스템의 I/O 응답을 기다리는 동안 DB 커넥션을 붙잡고 있게 되어 전체 시스템의 커넥션 풀이 마비된다.
- **롤백의 한계**: 로컬 DB가 롤백되더라도 이미 API 호출이 완료된 외부 시스템은 자동으로 롤백되지 않는다. (예: 포인트는 이미 **적립**되었으나 이후 결제가 실패하는 정합성 오류가 발생한다)
- **회색 영역(Grey Zone)의 발생**: 결제 요청 후 응답이 유실되면 성공도 실패도 아닌 상태가 되어, 시스템이 정상적인 후속 처리를 하지 못하는 교착 상태에 빠진다.
- **해결책**: 외부 연동 단계를 명확히 쪼개고, 각 단계의 성패를 로컬 DB에 **상태(Status)** 로 기록하며, 실패 시 보상하고 모르는 상태는 사후 배치로 수렴시킨다.

## 2. 회색 영역(Grey Zone)

- 외부 시스템 API 요청 후 응답이 오지 않는 경우(외부 시스템에서는 정상 응답을 보냈으나 네트워크 장애로 유실된 경우 등) 이를 즉시 실패로 단정 짓지 않고 지연 또는 확인 필요 상태로 처리하는 것이 좋다.
- **Connection Timeout**은 외부 서버가 요청 자체를 받은 적이 없으므로 재시도해도 안전하다.
- 반면 **Read Timeout**은 외부 서버가 요청을 받아 처리하는 도중 지연이 발생했거나, 처리를 완료하고 응답을 보냈으나 네트워크에서 유실된 경우를 모두 포함한다.
- 따라서 Read Timeout 발생 시 무작정 재시도를 감행하면 중복 처리가 될 수 있으며, 이처럼 성공과 실패를 명확히 알 수 없는 구간을 **회색 영역**이라고 한다.
- 이러한 회색 영역은 반드시 사후에 **보정(Reconciliation)** 단계를 거쳐야 한다.

## 3. 보정 배치(Reconciliation)

- 회색 영역에 머물러 있는 데이터를 사후에 최종 상태로 수렴시키는 과정이다.
- 외부 서버에 해당 요청 건의 상태를 조회하는 API를 호출하여, 외부 시스템에서 정상 처리되었다면 우리 서버의 상태도 완료로 변경하고 후속 프로세스를 진행한다.
- 외부 시스템에 이미 부작용(Side Effect, 실결제나 포인트 변동 등)이 일어났는지를 확인하여 보상 트랜잭션을 실행할지, 아니면 보정 배치를 통해 상태를 일치시킬지 결정한다.
- 보정 배치는 스프링(Spring) 내부 스케줄러부터 별도의 **스프링 배치(Spring Batch)** 등으로 구현할 수 있으며, **대사 작업**이라고 부르기도 한다.
